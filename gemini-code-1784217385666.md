# Instruksi Pembuatan Fitur Import Data Excel (Pelanggan & Kendaraan)

## Peran & Konteks Sistem
[cite_start]Bertindaklah sebagai Full-Stack Laravel Developer ahli[cite: 50]. Sistem ini adalah Customer Relationship Management (CRM) untuk dealer motor PT. [cite_start]Trijaya Motor[cite: 51]. 
[cite_start]Tech Stack yang digunakan meliputi PHP 8.3+, Laravel 13.x, Tailwind CSS, Alpine.js, dan Maatwebsite/Laravel-Excel[cite: 52].

## Objektif Tugas
[cite_start]Implementasikan fitur import data massal dari file Excel untuk Data Master (Pelanggan dan Kendaraan)[cite: 53]. [cite_start]Fitur ini harus memproses data, menghubungkan relasi antar tabel, dan secara otomatis membuat (*generate*) QR Code untuk pelanggan baru[cite: 54].

## Struktur Database Target
[cite_start]Lakukan interaksi dengan tabel-tabel berikut berdasarkan struktur ERD[cite: 55]:
* [cite_start]`pelanggan` (id_pelanggan, nama, alamat, no_hp, tanggal_daftar) [cite: 55]
* [cite_start]`kendaraan` (id_kendaraan, id_pelanggan, no_polisi, merek, tipe, tahun) [cite: 55]
* [cite_start]`qr_codes` (id_qr, id_pelanggan, kode_qr, status) [cite: 55]

## Format Kolom Excel (Flat Sheet)
[cite_start]File Excel yang diunggah akan menggunakan header berikut pada Baris 1[cite: 56]:
`nama_pelanggan` | `no_whatsapp` | `alamat` | `no_polisi` | `merek_kendaraan` | `tipe_kendaraan` | [cite_start]`tahun_kendaraan` [cite: 56, 57]

## Kebutuhan Backend (Laravel-Excel)
[cite_start]Buat kelas Import (contoh: `CustomerVehicleImport`) menggunakan package Maatwebsite/Laravel-Excel[cite: 57].
* [cite_start]Implementasikan traits: `WithHeadingRow`, `WithValidation`, dan `WithChunkReading` (chunk size: 500)[cite: 58].
* [cite_start]Aturan Validasi: `no_whatsapp` (required, string), `no_polisi` (required, string, hapus semua spasi sebelum disimpan), `nama_pelanggan`, `merek_kendaraan`, `tipe_kendaraan` (required)[cite: 59, 60].
* [cite_start]Logika Pemrosesan (Gunakan DB Transaction): Cari data pelanggan berdasarkan `no_hp` (mapping dari `no_whatsapp`)[cite: 60].
* [cite_start]Jika pelanggan ada, ambil `id_pelanggan`[cite: 61].
* [cite_start]Jika pelanggan tidak ada, buat data baru dan ambil `id_pelanggan` yang baru dibuat[cite: 62].
* [cite_start]Cari data kendaraan berdasarkan `no_polisi` (tanpa spasi)[cite: 63].
* [cite_start]Jika kendaraan tidak ada, buat data baru dan relasikan dengan `id_pelanggan`[cite: 64].
* [cite_start]Jika pelanggan baru saja dibuat, buat token UUID secara otomatis dan simpan ke tabel `qr_codes` dengan relasi ke `id_pelanggan`[cite: 65].
* [cite_start]Gunakan `SkipsOnFailure` untuk melewati baris yang error agar proses import tidak berhenti secara total[cite: 66].

## Kebutuhan Frontend (Blade + Alpine.js + Tailwind CSS)
[cite_start]Modifikasi halaman index Pelanggan (resources/views/...)[cite: 67].
* [cite_start]Tambahkan tombol "Import Data Excel" di dekat tombol "Tambah Pelanggan"[cite: 68].
* [cite_start]Buat modal tersembunyi menggunakan Alpine.js (`x-data`, `x-show`) yang terbuka saat tombol import diklik[cite: 69].
* [cite_start]Isi modal harus menyertakan input file untuk `.xlsx` dan `.csv`[cite: 70].
* [cite_start]Sediakan tombol/link "Download Template Excel" yang mengarah ke route pengunduhan template dummy[cite: 71].
* [cite_start]Tambahkan tombol Submit dengan *loading state* (ubah teks menjadi "Memproses..." atau gunakan spinner) untuk mencegah pengiriman data ganda[cite: 72].

## Kebutuhan Controller & Routing
[cite_start]Tambahkan metode pada Controller terkait (contoh: `CustomerController`)[cite: 73]:
* [cite_start]Metode `downloadTemplate()` untuk mengembalikan file template statis Excel[cite: 73].
* [cite_start]Metode `importExcel(Request $request)` untuk memvalidasi file, memanggil `Excel::import`, menangani exception, dan mengembalikan pesan *flash* sukses/error ke view[cite: 74].
* [cite_start]Berikan kode lengkap untuk route di `web.php`[cite: 75].
* [cite_start]Pastikan seluruh kode mematuhi standar *best practices* Laravel[cite: 76].
